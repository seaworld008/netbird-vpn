# 案例一：用 NetBird 替代 OpenVPN（基础生产场景）

## 1. 场景定义

- 原来团队使用 OpenVPN 进行远程接入。
- 目标：减小客户端维护、支持用户级访问控制、支持多平台。

## 2. 架构

- 1 台 NetBird 管控/控制节点（公网）
- 若干远程办公客户端（Windows/Mac/Linux/移动端）
- 一组内网资源节点（开发机、文件服务器、监控系统）

## 3. 实施步骤

### 步骤 1：部署服务

按 `docs/selfhosted/quickstart-modern.md` 部署，拿到 Dashboard。

### 步骤 2：建立“接入组”和“资源组”

1. 在 Access Control（访问控制）里创建组 `employees`
2. 将办公端设备添加到 `employees`
3. 对接企业网段，创建一个路由组 `office-resource`

### 步骤 3：创建网络路由

- 在管理后台创建路由规则：将企业网段（如 `10.20.0.0/16`）从网关节点宣告到 `employees`。
- 只给该组成员下发该路由。

### 步骤 4：改造访问控制

- 定义默认拒绝策略（必要）
- 放开到网段的 22/443/8443 等业务端口
- 如需更细化，按端口范围分段（TCP 3306、TCP 5432 等）

### 步骤 5：下发客户端

- 通过管理后台发放 Setup Key（按角色签发）
- 用户首登后，建议强制开启审计与设备状态（如支持系统版本、加密状态检查）

## 4. 与 OpenVPN 差异（运维视角）

- NetBird 无需手工维护每用户 config 文件。
- 支持按设备/用户/组做策略，不再以路由为中心。
- NAT/UDP 变更由客户端自动协商（依赖 STUN、Relay 和 NAT 打洞）。

## 5. 验收清单

- 远程端口 22/443 是否按策略可达
- 非授权组是否无法访问企业网段
- 客户端掉线后 30 秒内可自动重连（视网路而定）
