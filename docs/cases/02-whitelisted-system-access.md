# 案例二：企业内部白名单系统接入（按最小权限分发）

## 场景

企业内部有“白名单系统”（如风控白名单、审计系统）不允许公网访问，仅允许特定员工访问。

## 目标

- 只允许 `sec-audit-team` 和 `ops-team` 访问 `10.100.10.10`
- 仅放行 TCP 443、TCP 8443
- 非本组成员任何时候不可访问

## 前置条件

- 目标系统位于私有网段，例如 `10.100.10.0/24`
- 已完成 NetBird 客户端部署

## 实施步骤

### 1. 标记目标资源与用户

1. 在“组”中创建 `sec-audit-team`、`ops-team`
2. 绑定对应用户（或设备）到组
3. 将白名单系统所在网段（`10.100.10.10/32`）映射为网络资源

### 2. 配置分发路由

- 增加路由：源 = 上述两个组，目的网段 = `10.100.10.10/32`
- 选择已有能访问该网段的路由节点作为 Distribution Node

### 3. 配置访问策略

- 策略 A：`sec-audit-team` -> 资源组 `whitelist-system`
  - Protocol: TCP
  - Ports: `443,8443`
- 策略 B：`ops-team` -> 同上
- 默认策略改为“拒绝（或不放行）”

### 4. 验证

```bash
# 在授权客户端
curl -I https://10.100.10.10:443
nc -vz 10.100.10.10 8443

# 在非授权客户端，以上均应失败
```

### 5. 常见误区

- 路由没有下发到设备：检查 Distribution Group 与路由源组是否一致。
- 证书不受信：若内网 HTTPS 使用自签 CA，需在客户端分发证书或使用内网 CA 颁发证书。
- 网段重叠：复杂网段时优先明确掩码与最长前缀匹配。
