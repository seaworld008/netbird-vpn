# 阿里云安全组与端口加固（可直接复用）

## 一、基础端口

```text
TCP 80   (HTTP/ACME)
TCP 443  (HTTPS)
UDP 3478 (STUN)
UDP 443  (HTTP/3，可选，按反向代理配置)
```

端口作用说明：

- `80/tcp`：给 Let's Encrypt 或反向代理做证书申请、HTTP 跳转。
- `443/tcp`：用户访问 Dashboard、管理 API、浏览器登录的主入口。
- `3478/udp`：STUN，用来帮助客户端识别 NAT 和协商连接路径。
- `443/udp`：只有在启用 HTTP/3 时才需要。

## 二、本机防火墙示例（可选）

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 3478/udp
sudo ufw allow 443/udp comment 'Optional HTTP/3'
sudo ufw enable
```

> 生产环境建议再增加来源 IP 白名单：只允许办公网段访问管理/管理 API。

## 三、阿里云安全组配置示例

适用于绝大多数阿里云 ECS 新手场景：

| 方向 | 优先级 | 协议类型 | 端口范围 | 授权对象 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 入方向 | 1 | TCP | 80/80 | `0.0.0.0/0` | NetBird HTTP/证书 |
| 入方向 | 1 | TCP | 443/443 | `0.0.0.0/0` | NetBird HTTPS |
| 入方向 | 1 | UDP | 3478/3478 | `0.0.0.0/0` | NetBird STUN |
| 入方向 | 1 | UDP | 443/443 | `0.0.0.0/0` | 可选，HTTP/3 |

填写说明：

1. 在阿里云控制台进入 ECS 实例对应的“安全组”。
2. 选择“入方向”。
3. 授权对象公网开放时填 `0.0.0.0/0`。
4. 如果你有固定办公出口 IP，建议把 `443/tcp` 改成办公网 IP 白名单。
5. 添加完成后，再检查系统本机的 `ufw` 或其他防火墙是否也已放行。

## 四、安全基线建议

- 强制控制台管理员启用双因子
- 对外日志集中采集（控制面、客户端、网关）
- 变更必须走审批流程（尤其是策略变更）
- 对关键脚本与证书文件设置严格权限（root 归属）
